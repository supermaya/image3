"AI 생성 팝업의 모델 선택 기능을 제거하고, 워크플로우 기반 시스템으로 변경해줘. ComfyUI에서 생성된 이미지를 PixelSunday 화면에 즉시 표시하고 클라우드에 저장할 수 있게 기능을 수정해줘"

1.UI 수정:

기존의 모델 선택 드롭다운 대신, '워크플로우 선택' 드롭다운을 만들어줘. 워크플로우 선택을 누르면 해당 JSON 파일을 로드하도록 연결해줘.
aspect ratio 선택 : 16:9 (Panorama)와 9:16 (Slim Vertical) 중에 한가지를 선택할 수 있도록 해줘
비회원이 생성 버튼을 누르면 로그인 UI를 호출해줘.

2.동적 데이터 주입:

선택된 JSON 파일에서 class_type: "CLIPTextEncode" 노드를 자동으로 찾아 사용자의 프롬프트만 주입하고, 나머지 설정(모델, 시드 등)은 JSON의 설정을 그대로 유지하게 해줘.

3. 랜덤 시드 적용: 생성할 때마다 새로운 이미지가 나오도록 3번 노드(KSampler)의 seed 값을 실행 시마다 랜덤 숫자로 바꿔서 전송해줘.

4. 해상도: 해상도 선택 UI에서 지정한 aspect_ratio를 103번 노드에 적용해줘.

5. API 호출: 완성된 JSON 오브젝트를 http://192.168.219.42:8188/prompt로 POST 요청을 보내도록 설정해줘.

6. 이미지화일 생성 및 업로드 : 
   - 현재 로그인된 유저의 세션 정보에서 userName을 가져와줘.
   - 생성된 이미지는 ComfyUI 서버에서 가져온 뒤 Firebase Storage의 images/users/{userName}/gallery/ 경로에 저장해줘. 
   - 파일명은 중복을 피하기 위해 timestamp_random.png 형식을 사용해줘.
   - Firestore의 User 갤러리 컬렉션에 해당 이미지 URL을 추가해줘. 

7. users 컬렉션 내 해당 유저의 문서 안에 galleries라는 서브 컬렉션(또는 배열)을 만들어줘.
   생성된 이미지의 Storage URL, 생성 시간, 사용된 프롬프트, 워크플로우 이름을 이 문서에 함께 기록해줘.
   
8. 이미지 메타데이터 : 에이전트에게 "이미지를 저장할 때 메타데이터에 생성에 사용된 ZIT_VISVIS_API.json 설정값 일부를 태그로 저장해줘"

9. 권한 설정: Firebase 보안 규칙(Security Rules)에서 본인의 userName 폴더에는 본인만 접근(쓰기/읽기)할 수 있도록 설정해줘
 
10. 비동기처리 : 이미지를 가져올 때, 파일이 메모리에만 머물지 않고 Firebase에 업로드가 완료된 것을 확인한 후 UI에 띄우도록 **비동기 처리(Async/Await)**를 확실히 해줘
